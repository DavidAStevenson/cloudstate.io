Why?

Serverless computing makes developers more productive by taking care of operational concerns for them.

However, serverless functions fail to address the challenges associated with managing state.

CloudState sits at the confluence between serverless and state, making developers more productive by handling the operational and distribution concerns of state for them.

What?

State
Provide distributed state management patterns that ensure scalability and reslience.
Operations
Scale services intelligently and provide in depth observability.
Productivity
Allow developers to focus on their domain problem, rather than how state is managed.

CloudState is an architectural approach and implementation of that approach for providing distributed state management patterns suitable for serverless computing. Current supported and envisioned patterns include:

  • Event Sourcing
  • CRDTs
  • P2P messaging
  • CQRS read side projections

CloudState services can be written in any language that supports gRPC, with language specific libraries provided that allow idiomatic use of the patterns in each language. CloudState can be used either by itself, in combination with a service mesh, or it is in envisioned that it will be integrated with other serverless technologies such as Knative.
